# 노드플레이스 법률 검토 및 할 일

> **최종 업데이트**: 2026-02-22
> **참고**: 본 문서는 법률 자문이 아닌 사전 검토 자료입니다. 정식 운영 전 전문가 자문을 권장합니다.

---

## 1. 법률 검토 결과

### 1-1. 서비스 성격

노드플레이스는 파티 호스트가 이벤트를 개설하고, 고객이 온라인으로 참가 신청·입금하는 **예약 플랫폼**입니다.
고객은 각 호스트의 계좌로 직접 입금하며, 플랫폼은 결제를 중개하거나 대행하지 않습니다.

### 1-2. 적용 법령

| 법령 | 적용 대상 | 내용 |
|------|----------|------|
| 전자상거래법 | 온라인으로 서비스를 판매하는 자 | 사업자 정보 표시, 이용약관, 청약철회(환불) 규정 |
| 개인정보보호법 | 고객 개인정보를 수집하는 자 | 개인정보처리방침 공개, 책임자 지정, 동의 절차 |
| 통신판매업 신고 | 6개월간 거래 20회 이상 또는 1,200만 원 이상인 판매자 | 관할 구청에 신고, 신고번호 사이트에 표시 |

> **통신판매업 신고는 사실상 필수입니다.**
> 월 10회 파티 × 60명 기준으로 6개월간 거래 약 3,600회, 참가비 5만 원 기준 거래 금액 약 1억 8천만 원으로
> 면제 기준(20회 / 1,200만 원)을 즉시 초과합니다.

### 1-2-1. 유사 서비스 법적 준수 현황 (2026-02-22 조사)

| | 효스타임 | 모노파티 |
|--|---------|---------|
| 사업자등록번호 | 있음 (366-07-03250) | **없음** |
| 대표자 | 있음 (송준환) | **없음** |
| 통신판매업 신고번호 | **없음** | **없음** |
| 이용약관 | 링크 있음 | 빈 페이지 |
| 개인정보처리방침 | 링크 있음 | 빈 페이지 |

> **업계 현황**: 파티 업계는 인스타그램 DM 접수 → 개인 계좌 입금 방식이 일반적이며,
> 사업자 등록이나 세금 신고 없이 운영하는 경우가 많습니다.
> 웹사이트를 운영하는 업체도 통신판매업 신고를 하지 않고 있으나,
> 이는 업계가 법적 사각지대에 있는 것이지 합법인 것은 아닙니다.
>
> **리스크 증가 추세**:
> - 국세청의 SNS 소득 추적이 매년 강화되고 있으며, 일정 금액 이상의 계좌 입출금은 금융기관이 국세청에 자동 통보
> - 경쟁사나 불만 고객의 신고 한 건으로 무신고·탈세 즉시 적발 가능
> - 통신판매업 미신고 과태료 + 미신고 소득 가산세(최대 40%) 부과 대상
>
> 노드플레이스가 법적 요건을 선제적으로 갖추면 **고객 신뢰도 확보**와
> **호스트 보호**(정식 신고·운영 근거) 측면에서 경쟁 우위가 됩니다.

### 1-3. 사이트에 반드시 표시해야 할 사항

전자상거래법에 따라 사이트 초기화면(또는 하단)에 다음을 표시해야 합니다.

- **사업자 정보**: 상호, 대표자 성명, 사업장 주소, 전화번호, 사업자등록번호
- **통신판매업 신고번호** (해당 시)
- **이용약관**: 서비스 이용 조건, 결제·환불 정책, 면책 조항
- **개인정보처리방침**: 수집 항목, 이용 목적, 보유 기간, 책임자 성명·연락처

> 위반 시 과태료: 1차 100만 원, 2차 200만 원, 3차 500만 원

### 1-4. 수집하는 개인정보 항목

현재 노드플레이스가 고객으로부터 수집하는 정보:

| 항목 | 수집 목적 |
|------|----------|
| 이름 | 참가자 식별, 입금 확인 |
| 전화번호 | SMS 안내 발송 (승인·거절·QR) |
| 성별·나이 | 파티 참가 조건 확인 |
| 사진 | 호스트의 참가자 사전 확인 |
| 입금자명·은행 | 입금 확인·환불 처리 |
| 유입경로 | 마케팅 분석 (선택) |

## 2. 운영 방안

### 2-1. 1안: 호스트 개별 운영

> 각 호스트가 독립적으로 사업자 등록을 하고, 플랫폼은 단순 도구로 사용하는 방식입니다.

```
 호스트 A (개인사업자)     호스트 B (개인사업자)     호스트 C (개인사업자)
 - 사업자등록 각자         - 사업자등록 각자         - 사업자등록 각자
 - 파티 개설·고객 관리     - 파티 개설·고객 관리     - 파티 개설·고객 관리
 - 직접 입금 수령          - 직접 입금 수령          - 직접 입금 수령
 - 매출·세금 각자 신고     - 매출·세금 각자 신고     - 매출·세금 각자 신고
         │                         │                         │
         └─────────────────────────┼─────────────────────────┘
                                   │
                          nodeplace.kr 이용
                  (시범 운영 기간 무상, 정식 전환 시 협의)
```

**사이트 표시 방법**
- 사업자 정보: 호스트 중 **대표 1명**의 정보를 표시하거나, 각 호스트 프로필에 개별 표시
- 개인정보 책임자: 대표 호스트 또는 호스트별 지정

**장점**
- 별도 법인 참여 불필요, 호스트끼리 자율 운영
- 수익 정산이 단순 (각자 입금받으므로 정산할 것이 없음)

**단점**
- 사업자 정보 표시가 복잡 (사이트는 하나인데 사업자가 여러 명)
- 개인정보 수집 주체가 불명확 (누가 책임자인지 애매)
- 호스트 추가·탈퇴 시 사업자 정보 변경 번거로움
- 이용약관·개인정보처리방침의 법적 주체가 불분명

### 2-2. 2안: 공동 법인 설립 (권장)

> 호스트들이 함께 새로운 법인을 설립하여 노드플레이스를 운영하는 방식입니다.
> 플랫폼 개발·운영은 외부 개발사에 위탁합니다.

```
┌──────────────────────────────────────┐
│     공동 법인 (호스트들이 공동 설립)    │
│  - nodeplace.kr 운영 주체             │
│  - 사업자 정보 표시 (법인 명의)        │
│  - 이용약관·개인정보처리방침 주체       │
│  - 개인정보 수집·처리 책임             │
│  - 인프라 비용 부담                    │
│                                      │
│  구성원: 호스트 A + B + C (지분 협의)   │
└────────┬─────────────────┬───────────┘
         │                 │
         │ 파티 운영        │ 개발·운영 위탁
         ▼                 ▼
  호스트 A, B, C        개발사 (외부)
  - 파티 개설·관리      - 플랫폼 개발·유지보수
  - 고객 입금 수령      - 서버·인프라 관리
  - 고객 응대           - 기술 지원
```

**역할 분담**

| | 공동 법인 | 개발사 (외부 위탁) |
|--|----------|-------------------|
| 사업자등록 | 공동 법인 (신규) | 기존 법인 |
| 사이트 사업자 정보 | **공동 법인 정보 표시** | — |
| 이용약관 | 공동 법인 명의로 작성·공개 | — |
| 개인정보처리방침 | 공동 법인 명의, 책임자 = 대표이사 | 개인정보 처리 수탁자로 명시 |
| 통신판매업 | 매출 기준 초과 시 공동 법인 명의로 신고 | — |
| 파티 운영·환불 | 법인 소속 호스트가 각자 담당 | — |
| 인프라 비용 | 법인이 부담 | 위탁 수수료 수령 |
| 플랫폼 개발·운영 | — | 개발·유지보수·장애 대응 |

**사이트 하단 표시 예시**

```
상호: ○○○(공동법인명)  |  대표: ○○○  |  사업자등록번호: 000-00-00000
주소: ○○시 ○○구 ○○로 00  |  전화: 000-0000-0000
통신판매업 신고번호: 제0000-○○○○-0000호
이용약관  |  개인정보처리방침
```

**법인 설립 시 결정할 사항**

| 항목 | 내용 |
|------|------|
| 법인 형태 | 주식회사 또는 유한회사 (소규모는 유한회사가 간편) |
| 지분 구조 | 호스트 3명의 지분 비율 협의 |
| 대표이사 | 호스트 중 1명 또는 공동대표 |
| 법인 계좌 | 인프라 비용·문자비용 등 공동 경비 출금용 |
| 호스트 입금 | 각 호스트 **개인 계좌**로 고객이 직접 입금 (기존과 동일) |

**셀프 법인설립 (온라인)**

> [온라인 법인설립시스템(startbiz.go.kr)](https://www.startbiz.go.kr)에서 법무사 없이 직접 설립 가능합니다.
> 최저자본금 제도가 폐지되어 자본금 100원 이상이면 설립할 수 있으며, 소요 기간은 약 4~7일입니다.

| 항목 | 비용 |
|------|------|
| 등록면허세 + 교육세 | 약 13.5만 원 (자본금 2,800만 원 이하) |
| 등기수수료 | 2만 원 |
| 잔고증명서 | 약 3천 원 |
| **셀프 합계** | **약 16만 원** |

> 필요한 것: 대표자 공동인증서, 법인인감, 자본금 통장, 스캐너

**청년창업지원 프로그램 활용**

> 법인 설립 전에 정부 창업지원 프로그램에 먼저 신청하면
> 법인 설립 비용·인프라 비용·마케팅 비용 등을 지원금으로 충당할 수 있습니다.
> 노드플레이스는 이미 작동하는 IT 플랫폼(MVP)이 있어 심사 시 유리합니다.

| 프로그램 | 지원 금액 | 대상 | 신청 시기 |
|----------|----------|------|----------|
| 예비창업패키지 | 최대 8천만 원 | 사업자 미등록 예비창업자 (나이 무관) | 연초 1~3월 |
| 초기창업패키지 | 최대 1억 원 | 창업 3년 이내 기업 | 연초 1~3월 |
| 창업중심대학 청년 지원 | 최대 1억 원 | 만 29세 이하 생애최초 | 연초 |
| 청년전용 창업자금 (융자) | 최대 1억 원 | 만 39세 이하 | 상시 |

> **추천 전략**: 예비창업패키지는 사업자 **미등록** 상태에서 신청해야 하므로,
> 법인 설립 전에 먼저 신청 → 선정 시 지원금으로 설립 비용 충당.
> 미선정 시에도 셀프 법인설립(약 16만 원)으로 부담 없이 진행 가능.
>
> 상세 정보: [K-Startup 창업지원포털](https://www.k-startup.go.kr/)

**장점**
- 사업자 정보·개인정보 책임 주체가 **공동 법인 하나로 명확**
- 호스트 전원이 법인 구성원이므로 의사결정이 투명
- 호스트 추가·탈퇴 시에도 법인은 유지되어 사이트 표시 변경 불필요
- 개발사는 위탁 관계로만 참여하여 운영 책임 부담 없음
- 인프라 비용을 법인 경비로 처리 가능
- 청년창업지원 프로그램 활용 시 초기 비용 대부분 지원 가능

**단점**
- 법인세 신고 등 세무 관리 필요 (세무사 위탁 권장)
- 지분·의사결정 구조에 대한 사전 합의 필요

## 3. 할 일 목록

### 3-1. 시범 운영 (현재)

> 1차 요구사항 구현 완료. 호스트 계정 세팅 후 테스트 및 피드백 단계로 진입합니다.

**호스트 계정 세팅**
- [ ] 호스트별 이메일 주소·전화번호 수집
- [ ] 관리자 계정 생성 (어드민 페이지)
- [ ] 호스트 프로필 설정 (닉네임, 소개, 프로필 사진, SMS 발신번호, 입금계좌)
- [ ] 각 호스트에게 어드민 접속 안내 및 로그인 테스트

**호스트 테스트 및 피드백**
- [ ] 파티 생성·수정·삭제 테스트
- [ ] 고객 신청 접수 → 승인/거절 → SMS 발송 테스트
- [ ] 입금 확인·참석 관리 흐름 테스트
- [ ] 피드백 수집 및 정리

**운영 방안 확정**
- [ ] 1안(호스트 개별) 또는 2안(공동 법인) 선택

### 3-2. 콘텐츠·기능 보완

**콘텐츠 조정**
- [ ] About 페이지 내용 검토 및 수정 (소개 문구, 이미지 등)
- [ ] 메인 페이지 안내 문구 검토

**기능 보완**
- [ ] 피드백 기반 UI/UX 개선
- [ ] 파티별 참여옵션 관리 (가격·시간 커스터마이징)
- [ ] 금액 수동 수정 기능 (현장 결제 등)

### 3-3. 법인 설립 (2안 선택 시)

- [ ] 청년창업지원 프로그램 신청 검토 (예비창업패키지 등, 법인 설립 전에 신청)
- [ ] 지분 구조·대표이사 합의
- [ ] 셀프 법인설립 (온라인 법인설립시스템, 약 16만 원)
- [ ] 법인 사업자등록
- [ ] 법인 통장 개설 (공동 경비용)
- [ ] 통신판매업 신고 (면제 기준 즉시 초과 예상, 사실상 필수)
- [ ] 개발사와 개발·운영 위탁 계약 체결

### 3-4. 보안 점검

- [ ] 입력값 검증 (XSS, SQL 인젝션 등)
- [ ] 인증·권한 검증 (API 접근 제어)
